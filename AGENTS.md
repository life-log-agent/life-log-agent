# AGENTS.md — life-log-agent

이 문서는 에이전트(Claude Code 등)가 이 저장소에서 작업할 때 따르는 **운영 규칙**이다.
아키텍처·명령어·컨벤션의 상세는 `CLAUDE.md`, 확정 설계는 `docs/superpowers/specs/2026-05-31-life-log-platform-design.md`를 따른다.

## 프로젝트 개요

`life-log-agent`는 **실제로 운영 가능한 AI 라이프로그 서비스**다. 사용자가 갤러리 캡처·사진·음성메모·PDF를 업로드하면, AI가 내용을 추출·자동 분류하고, 나중에 **자연어로 검색·질의(RAG)** 할 수 있게 한다. 폰은 업로드 기기, PC는 조회·검색 기기이며 같은 클라우드 계정을 공유한다.

## 기술 규칙

- **프레임워크는 이전 대화(브레인스토밍·스펙)를 기반으로 한다:**
  - 프론트엔드: **React PWA** (Vite + TypeScript) — 반응형·설치형, 데스크톱·iOS·안드로이드 공용.
  - 백엔드: **FastAPI** (순수 JSON API).
  - 데이터/저장/인증: **Supabase** (Postgres + pgvector + Storage + Auth).
  - 멀티모달 LLM/임베딩/STT: **Clova Studio HCX-005** + Clova Embedding + CLOVA Speech.
- **저장은 클라우드(Supabase) 기준으로 한다.** 단, PWA 오프라인 셸 캐시·임시 초안 등 보조 용도의 클라이언트 저장은 허용한다.
- **외부 의존은 `app/integrations/` 래퍼를 통해서만 호출한다.** 서비스·라우터에서 외부 SDK/REST를 직접 부르지 않는다.
- **화면은 복잡한 칸반 보드가 아니라 단계형(가이드형) 흐름 GUI로 만든다.** 업로드 → 처리 상태 → 분류 결과 → 검색을 명료한 단계로 보여 주고, 한 화면에 정보를 욱여넣지 않는다.
- **모바일에서 텍스트가 겹치거나 잘리지 않게 한다.** PWA는 폰이 1차 사용 환경이므로 반응형 레이아웃과 가독성을 우선한다.

## 하네스 규칙

- **불명확하면 Superpowers `brainstorming`으로 질문**해 요구사항을 먼저 좁힌다. 추측으로 구현하지 않는다.
- **여러 관점이 필요하면 OMC `/team`** 으로 기획자·구현자·검토자 의견을 나눈다. (설계 트레이드오프, 리스크 큰 변경 등)
- **구현 전에는 `PLAN.md`를 만든다.** 단계별 실행 계획 없이 코드 작성에 들어가지 않는다.
- **완료 전에는 `VERIFY.md`에 실제 확인 결과(실행 명령·출력·결과)를 남긴다.** 증거 없이 완료를 주장하지 않는다.
- **확인하지 못한 것은 성공한 것처럼 쓰지 않는다.** 테스트가 실패했거나 단계를 건너뛰었으면 그대로 보고한다.

## 금지

- **API 키·토큰을 파일에 하드코딩하지 않는다.** Supabase service key·Clova API 키는 `config.py` settings / 환경변수로만 읽고, `.env`는 커밋하지 않는다(`.env.example` 제공). service role 키는 백엔드 전용, 프론트엔드에는 anon 키만 노출한다.
- **최종 프로젝트를 한 번에 완성하려 하지 않는다.** 단계(질문→팀 검토→계획→구현→검증)를 건너뛰고 일괄 구현하지 않는다. 증분으로 만들고 단계마다 검증한다.
- **불필요한 라이브러리를 추가하지 않는다.** 표준·기존 의존으로 해결 가능하면 새 패키지를 넣지 않는다. 추가가 필요하면 이유를 먼저 밝힌다.
