# PPT_GAP_CHECK.md — 역할별 갭 점검 (Day8 발표)

> `PPT_EVIDENCE.md`(기준 커밋 `7c09115`)를 6개 역할 관점으로 검토하고 **빠진 자료와 보완 방법만** 정리한다.
> 이미 충분한 항목은 적지 않는다. 각 갭에 **무엇이 없는지 / 어떻게 채우는지 / 우선순위**를 표기.
> 우선순위: 🔴 발표 전 필수 · 🟡 있으면 설득력↑ · ⚪ 시간 남으면

## 1. 프로젝트 매니저 — "목적과 사용자가 분명한가"

| 갭 | 보완 방법 | 우선순위 |
|----|-----------|----------|
| **Before/After 한 장이 없음** | "갤러리에 쌓인 캡처(Before) → 자연어로 바로 찾음(After)" 대비 1장. 텍스트로도 충분: *그 화장품 캡처 어디 갔지? → "사고 싶던 화장품 캡처 모아줘" 한 줄로 끝* | 🟡 |
| **데모에서 보여줄 1개 시나리오가 끝까지 정해져 있지 않음** | 대표 시나리오 1개를 **업로드→처리완료→검색→답변+근거**까지 풀스텝으로 고정(예: 제주 흑돼지 사진 3장 → "제주 흑돼지 식당 어디였지?"). 발표 데모 = 이 한 줄 | 🔴 |
| **MVP 범위 경계(비목표) 미표기** | "이번 범위 밖" 1줄: 음성 STT·PDF·자동 갤러리 수집(브라우저 제약)은 제외 → 범위를 좁혀 완성했다는 인상 | 🟡 |
| **현재 상태(완성도) 한 줄 없음** | "실서버 배포 완료 + 백엔드 15 tests pass" 같은 진척 상태를 표지에 1줄(부록엔 있음 → 본문 PM 슬라이드로 끌어올리기) | ⚪ |

## 2. AI 기능 엔지니어 — "AI 입력/출력/실패 상태가 보이는가"

| 갭 | 보완 방법 | 우선순위 |
|----|-----------|----------|
| **실제 입출력 예시(샘플 1쌍)가 없음** | 데모 이미지 1장 → 실제 `describe_image` 출력(`OCR:…/설명:…`) → `classify` JSON(`{category,tags,place,summary}`) 한 쌽 캡처. 표 설명보다 **실물 예시 1쌍**이 가장 강력 | 🔴 |
| **AI 실패 상태가 §4에 안 보임** | 코드엔 있으나 슬라이드엔 없음 → 한 줄씩 명시: ① JSON 파싱 실패 시 `{기타,…}`/`{answered:false}` 폴백(`llm.py:122,164`) ② 타임아웃 60s(`llm.py:15`) ③ OCR 텍스트 없으면 임베딩 skip(`index.py:50`) ④ 파이프라인 예외 → `status=failed` + 재시도 | 🔴 |
| **답변 근거(evidence) 출력 형태 미설명** | RAG 출력이 `{answer, evidence[], query}` 구조이고 evidence에 썸네일·날짜·카테고리·점수가 함께 온다는 점 1줄(`SearchResponse`) → "출처 있는 답변"을 화면과 연결 | 🟡 |
| **임베딩 차원 일치 근거** | `embedding_dim=1024`(config) ↔ pgvector 컬럼 차원 ↔ Clova v2 출력이 일치해야 검색이 됨 — 이 정합성 1줄 | ⚪ |

## 3. Ncloud 설계자 — "Ncloud 서비스를 왜 썼는지 설명되는가"

| 갭 | 보완 방법 | 우선순위 |
|----|-----------|----------|
| **아키텍처 연결도(다이어그램) 이미지가 없음** | 텍스트 흐름만 있음 → 박스 다이어그램 1장: `PWA → (FastAPI on NCloud) → Clova HCX-005 / Clova Embedding`, 옆에 `Supabase(Storage·Postgres+pgvector·Auth)`. 화살표에 "Signed URL", "임베딩", "JWT 검증" 라벨 | 🔴 |
| **대안 대비 선택 이유가 약함** | "왜 Clova인가"를 1줄: 한국어 OCR·이해 품질 + 국내 인프라/과제 요건 정합 + 비전 멀티모달이라 OCR 엔진 분리 불필요(파이프라인 단순화). 타 LLM 대비 차별점으로 | 🟡 |
| **역할 분리(매니지드 vs 셀프호스팅) 의도 미설명** | "무거운 데이터 플랫폼은 Supabase 매니지드, 앱 구동만 NCloud 서버" — 운영 부담을 줄인 의도적 설계라는 1줄(현재는 사실만 나열) | 🟡 |

## 4. 디자인 팀장 — "화면이 실제 앱처럼 보이는가"

| 갭 | 보완 방법 | 우선순위 |
|----|-----------|----------|
| **실제 스크린샷이 0장 (가장 큰 갭)** | `PPT_EVIDENCE.md`는 화면을 **표로만** 나열 → 실제 구동 캡처가 없으면 "앱처럼" 안 보임. 실서버/로컬에서 핵심 4~5장 캡처: ①홈(타임라인) ②업로드(다중선택) ③처리상태 ④검색+답변+근거 ⑤상세. **모바일 폭으로 캡처** | 🔴 |
| **빈/로딩/실패 상태 캡처 누락** | 화면 목록엔 `HomeEmpty·Processing·ProcessingFailed·SearchEmpty`가 있으나 캡처 없음 → 빈 상태·처리중·실패+재시도 1장씩이면 "상태를 챙긴 앱" 인상 | 🟡 |
| **디자인 톤 근거가 화면과 안 연결됨** | "모바일 우선·하단 고정 업로드 버튼·단계형 흐름"을 실제 캡처에 화살표로 표시(말로만 X). `docs/DESIGN_*.md` 기준과 1:1 매칭 | 🟡 |
| **데모 영상/GIF 없음** | 클릭 흐름 10~20초 GIF 1개면 정적 캡처보다 "동작하는 앱" 증명에 강함 | ⚪ |

## 5. 기능 검증 팀장 — "테스트와 오류 수정 기록이 있는가"

| 갭 | 보완 방법 | 우선순위 |
|----|-----------|----------|
| **E2E(실제 외부 연동) 검증 증거 없음** | 자동 테스트는 `integrations/` 모킹이라 **실제 HCX-005 연동 동작은 증명 안 됨** → 실서버에서 업로드 1건이 `ready`까지 간 화면/`GET /items` 응답 1개 캡처로 보완(= 실연동 증거) | 🔴 |
| **프론트 빌드/lint 결과 미기재** | 백엔드만 `15 passed`/ruff. 프론트 `pnpm build` 성공 + `pnpm lint` 결과 1줄 추가 | 🟡 |
| **mypy 결과가 빠짐(정직성)** | 현재 mypy는 `app/__init__.py` 부재로 모듈명 충돌→미실행. "mypy 미실행(LOW, 알려진 사유)"로 정직하게 1줄. 숨기면 질문 시 신뢰 손상 | 🟡 |
| **수동 검증 체크리스트 부재** | 데모 직전 돌릴 수동 시나리오 5줄(로그인→업로드→ready→검색→근거 표시→실패 재시도). 통과 여부 ✅ 표기 | ⚪ |

## 6. 보안 팀장 — "민감정보가 들어갈 위험이 없는가"

| 갭 | 보완 방법 | 우선순위 |
|----|-----------|----------|
| **스크린샷에 PII 들어갈 위험(가장 현실적)** | 4번에서 캡처를 추가하는 순간 위험 발생 → 캡처 규칙 명문화: **데모 전용 계정·합성 이미지만**, 캡처본에서 이메일/JWT/실 IP 마스킹, 개발자도구 닫고 캡처 | 🔴 |
| **RLS 미적용 상태가 §9에 없음** | 보안 패치 코드는 배포됐으나 **Supabase RLS/Storage 정책 SQL은 아직 미적용**(후속). 발표에 "보안 강화 후속 1건(RLS 적용 예정)"으로 정직하게 1줄 — 숨기지 말 것 | 🟡 |
| **git 히스토리에 과거 실 IP가 남음** | 현재 파일은 `<SERVER_IP>`로 마스킹됐으나 **과거 커밋 히스토리엔 실 IP가 남아있음**. 공개 저장소면 발표 후 정리 검토(현 IP가 유효 자산이면 더 주의). 발표 중 `git log` 화면공유 금지 | 🟡 |
| **발표 직전 1분 점검표 부재** | 화면공유 전 체크: ☐ `.env`/터미널 env 안 보임 ☐ 개발자도구 닫힘(JWT) ☐ 실 IP/호스트 안 보임 ☐ 데모 계정·합성 이미지 ☐ `git log`/`/docs` 토큰 미노출 | 🔴 |
| **이 문서 2종(EVIDENCE/GAP)의 커밋 여부 판단** | 둘 다 실 비밀값은 없으나 운영 디테일 포함 → 공개 저장소면 커밋 전 검토. 현재 `docs/ppt-day8` 브랜치 작업물 | ⚪ |

## 종합 — 🔴 발표 전 필수만 (이것만 채우면 발표 가능)

1. **실제 화면 캡처 4~5장** (홈·업로드·처리상태·검색+근거·상세) — *디자인/검증/PM 동시 충족*
2. **AI 실제 입출력 샘플 1쌍** (이미지→OCR/설명→분류 JSON)
3. **AI 실패 상태 4줄** (JSON 폴백·타임아웃·빈OCR skip·failed 재시도)
4. **아키텍처 연결도 1장** (PWA→FastAPI/NCloud→Clova / Supabase)
5. **E2E 실연동 증거 1컷** (업로드→`ready`→검색 성공)
6. **데모 시나리오 1개 확정** (업로드~답변 풀스텝)
7. **보안 1분 점검표 + 캡처 PII 규칙** (스크린샷 추가에 따른 신규 위험)

> 🟡/⚪는 시간에 따라. 위 7개는 대부분 **새 자료 생성이 아니라 이미 동작하는 걸 캡처/한 줄 추가**라 30~60분이면 채워진다.
