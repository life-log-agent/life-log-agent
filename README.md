# life-log-agent

**실제로 운영 가능한 AI 라이프로그 서비스** — 갤러리 캡처·사진·음성메모·PDF를 업로드하면 AI가 내용을 추출·자동 분류하고, 나중에 **자연어로 검색·질의(RAG)** 할 수 있게 한다.

이 프로젝트는 동시에 **AI 보조 개발 실습의 교보재**다. 학생은 긴 코드를 직접 짜지 않는다. 대신 **Claude Code에게 프롬프트로 작업을 지시**하면서, "질문 → 팀 검토 → 계획 → 구현 → 검증" 워크플로우로 하나의 실제 서비스를 완성하는 과정을 체험한다. 각 단계에서 OMC와 Superpowers를 언제 쓰는지 함께 익힌다.

- 아키텍처·명령어·컨벤션: `CLAUDE.md`
- 에이전트 운영 규칙: `AGENTS.md`
- 확정 설계: `docs/superpowers/specs/2026-05-31-life-log-platform-design.md`

---

## 프로젝트 목표

1. **실제 운영 가능한 서비스 구축** — 데모가 아니라, 사용자가 실제로 쓸 수 있는 라이프로그 서비스를 만든다.
   - 프론트엔드: **React PWA**(반응형·설치형, 데스크톱·iOS·안드로이드 공용)
   - 백엔드: **FastAPI** + **Supabase**(Postgres+pgvector·Storage·Auth) + **Clova Studio HCX-005**(멀티모달 LLM)
   - 폰은 업로드 기기, PC는 조회·검색 기기 (같은 클라우드 계정 공유)
2. **5단계 워크플로우로 진행** — 주제를 다음 순서로 정리하고 실행한다.

   | 단계 | 하는 일 | 결과물 |
   |------|---------|--------|
   | ① 질문 | 목적·제약·성공 기준을 좁히는 질문을 던진다 | 정리된 요구사항 |
   | ② 팀 검토 | 여러 관점(설계·리스크·사용자)에서 접근안을 비교한다 | 추천안 + 트레이드오프 |
   | ③ 계획 | 단계별 구현 계획을 세운다 | `PLAN.md` |
   | ④ 구현 | 프롬프트로 작업을 지시해 코드를 만든다 | 동작하는 코드 |
   | ⑤ 검증 | 실제로 돌려 보고 결과를 확인한다 | `VERIFY.md` (증거 기반) |

3. **OMC vs Superpowers 안내** — 각 단계에서 **어떤 도구를 언제 쓰는지** 보여 준다.

   | 단계 | Superpowers | OMC |
   |------|-------------|-----|
   | ① 질문 | `brainstorming` (요구사항 도출) | — |
   | ② 팀 검토 | — | `/team`, `architect`, `critic` |
   | ③ 계획 | `writing-plans` (PLAN.md) | `planner`, `ralplan` |
   | ④ 구현 | `executing-plans`, `TDD` | `executor`, `autopilot`, `ultrawork` |
   | ⑤ 검증 | `verification-before-completion`, `requesting-code-review` | `verifier`, `ultraqa` |

   기준: **간단·단일 파일은 기본 Claude Code, 애매하면 Superpowers brainstorming, 다단계·역할 분담은 OMC Team**으로 라우팅한다.

4. **프롬프트 주도 흐름** — 학생은 코드를 손으로 길게 작성하지 않는다. "이렇게 만들어 줘"를 Claude Code에 지시하고, 산출물(스펙·계획·코드)을 검토·검증하는 역할에 집중한다.

## 수업 위치

이 프로젝트는 **AI 보조 소프트웨어 개발(에이전트 활용) 실습 과정**의 후반 캡스톤 모듈에 해당한다.

- **선행(이 수업 전):** Claude Code 기본 사용법, 스킬/슬래시 커맨드 개념, OMC·Superpowers 설치.
- **이 수업(현재 모듈):** "아이디어 → 실제 운영 가능한 서비스"까지의 **전체 워크플로우**를 한 사이클로 직접 돌려 본다. `life-log-agent`가 그 결과물이자, 학생이 같은 절차로 만들어 보는 실습 대상이다.
- **후행(이 수업 후):** 학생이 **자신만의 주제**를 골라 동일한 5단계로 캡스톤 프로젝트를 수행한다.

> 핵심 학습 목표: "코드를 얼마나 잘 짜는가"가 아니라 **"에이전트에게 어떤 순서로, 어떤 도구로, 어떻게 지시하고 검증하는가"** 를 익히는 것.

## 최종 산출물

1. **실제 동작하는 AI 라이프로그 서비스** (`life-log-agent`)
   - React PWA 프론트엔드 + FastAPI 백엔드 + Supabase + Clova HCX-005.
   - 업로드(갤러리 다중 선택) → 자동 분류 → 자연어 검색까지 end-to-end 동작.

2. **워크플로우 문서 세트** (워크플로우 ①~⑤가 만들어 낸 산출물)
   - `README.md` — 본 문서(목표·수업 위치·산출물).
   - `CLAUDE.md` — Claude Code가 따를 아키텍처·명령어·컨벤션 가이드.
   - `AGENTS.md` — 에이전트 운영 규칙(기술·하네스·금지).
   - `docs/superpowers/specs/2026-05-31-life-log-platform-design.md` — 확정 설계 스펙(①~③의 결과).
   - `PLAN.md` — 구현 계획(③, 구현 착수 시 생성).
   - `VERIFY.md` — 검증 결과(⑤, 완료 주장 전 생성).

3. **발표/회고 자료**
   - 각 단계에서 어떤 프롬프트를 썼고, OMC와 Superpowers 중 무엇을 왜 선택했는지 정리한 기록.

> 진행 범위: 현재는 **문서·설계 단계**까지 완료됐고, 구현은 `PLAN.md` 작성 이후 단계적으로 진행한다. 자동 갤러리 수집·네이티브 앱 등은 설계 스펙 §2.2에 따라 범위 밖이다.
